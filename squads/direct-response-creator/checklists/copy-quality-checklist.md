# Copy Quality Checklist — Direct Response Creator Squad

Use este checklist antes de considerar qualquer output pronto para uso.  
Execute via `@judge *review-all` ou manualmente marcando cada item.

---

## Pré-produção (núcleo)

### Avatar (Seção 3)
- [ ] Mínimo 20 campos preenchidos
- [ ] Nenhum campo com linguagem genérica ou abstrata
- [ ] Dores escritas como o avatar descreveria para um amigo
- [ ] Verbal triggers identificados e específicos
- [ ] Calibrado para o perfil de mercado ativo

### Oferta (Seção 4)
- [ ] Nome da oferta selecionado e aprovado
- [ ] Módulos descritos com linguagem de benefício, não feature
- [ ] 3 bônus com valor percebido maior que o preço principal
- [ ] Garantia escrita de forma persuasiva (não legalês)
- [ ] 3 depoimentos com nome específico + resultado com número + contexto antes

### USP (Seção 5)
- [ ] Nova categoria claramente definida e única
- [ ] Unique Mechanism com nome criativo e intrigante
- [ ] Resultados esperados com timeline e números (onde suportado)
- [ ] Diferenciação clara das "soluções falsas"

### Upsell (Seção 6)
- [ ] Nenhum order bump conflita com bônus da oferta principal
- [ ] Cada upsell resolve o próximo problema lógico após a compra
- [ ] Preços de order bumps menores que oferta principal

---

## Produção

### TSL — Carta de Vendas (Seção 7)
- [ ] Contagem de palavras ≥ 2.000
- [ ] 9 seções presentes e completas
- [ ] Hook emocional na abertura (não apresenta produto de cara)
- [ ] 3-4 objeções antecipadas e destruídas
- [ ] Unique Mechanism mencionado na Seção 5 da TSL
- [ ] Cada bônus apresentado com valor percebido
- [ ] CTA específico e direto (não "saiba mais")
- [ ] Ausência de AI-writing (adjetivos empilhados, frases genéricas)
- [ ] Linguagem nativa do mercado-alvo

### Headlines (Seção 8)
- [ ] Exatamente 20 headlines
- [ ] Mínimo 5 frameworks distintos
- [ ] Nenhum ângulo repetido com palavras diferentes
- [ ] Pelo menos 2 headlines com número específico
- [ ] Pelo menos 1 headline com Unique Mechanism

### Scripts de Vídeo (Seção 9)
- [ ] Cada script ≤ 165 palavras (EN-US) / 150 (PT-BR) / 155 (ES)
- [ ] Hook nos primeiros 3 segundos
- [ ] Estrutura do ângulo respeitada (não mistura seções)
- [ ] CTA específico (não "clique aqui para saber mais")
- [ ] Notas de direção incluídas
- [ ] Linguagem falada, não escrita

### Emails (Seção 10)
- [ ] Assunto ≤ 50 caracteres
- [ ] Assunto sem ponto de exclamação
- [ ] Abertura sem "Olá / Oi / Prezado / Espero que esteja bem"
- [ ] Exatamente 1 CTA por email
- [ ] Estrutura do template respeitada
- [ ] Linguagem nativa do mercado-alvo

### FAQ (Seção 11)
- [ ] Entre 5 e 10 perguntas
- [ ] Perguntas na linguagem do avatar (não de SAC corporativo)
- [ ] Todas as respostas sem exclamação
- [ ] Cada resposta termina com reforço de valor ou urgência
- [ ] Cobre pelo menos: risco, identidade e ceticismo

---

## Qualidade Geral (Anti-Devaneio)

- [ ] Nenhuma estatística sem fonte no material de input
- [ ] Nenhum resultado extraordinário sem evidência (`[PROVA NECESSÁRIA]` como placeholder se necessário)
- [ ] Nenhum claim de "o melhor", "o único", "revolucionário" sem substância
- [ ] Nenhum AI-writing detectado em todo o material

---

## Consistência Interna

- [ ] Avatar (Seção 3) → Copy: as dores mencionadas no copy existem no avatar
- [ ] USP (Seção 5) → TSL: o Unique Mechanism aparece na carta de vendas
- [ ] Oferta (Seção 4) → Emails: bônus mencionados nos emails batem com a oferta
- [ ] Tom de voz consistente entre TSL, emails, scripts e FAQ
